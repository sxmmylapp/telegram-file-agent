<task type="auto">
  <name>Task 1: Install PDFKit and create PDF generation service</name>
  <files>package.json, src/services/pdf-generator.ts</files>
  <action>
Install pdfkit and @types/pdfkit:
```bash
npm install pdfkit && npm install -D @types/pdfkit
```

Create `src/services/pdf-generator.ts` with the following structure:

1. **PdfMetadata interface**: `{ sourceFileName: string; generatedAt: Date }`

2. **parseSummarySections(markdown: string): SummarySection[]** — Parse Claude's structured output into typed sections. Each section has a `heading` (from `## Heading` lines) and `items` (from `- ` bullet lines and paragraph text). Handle edge cases: blank lines between sections, continuation text without bullet prefix, empty sections.

3. **renderFormattedText(doc, text)** — Handle inline bold (`**text**`) and italic (`*text*`) by splitting on `/(\*\*[^*]+\*\*|\*[^*]+\*)/` regex, switching between `Helvetica-Bold`, `Helvetica-Oblique`, and `Helvetica` fonts using the `continued` option. CRITICAL: always reset font to `Helvetica` after the last segment to avoid font bleed.

4. **renderHeader(doc, metadata)** — Render a clean header at the top of the first page:
   - "EXECUTIVE SUMMARY" in Helvetica-Bold, 20pt, dark color (#1a1a1a)
   - Source filename in Helvetica, 11pt, gray (#666666)
   - Generated date formatted as "February 19, 2026" in Helvetica, 10pt, gray
   - A horizontal rule (thin line from left margin to right margin) below the header
   - 20pt vertical space after the rule

5. **renderSection(doc, section)** — For each SummarySection:
   - Section heading in Helvetica-Bold, 14pt, dark blue-gray (#2c3e50)
   - 8pt space after heading
   - Each item as a bullet point: render a bullet character ("•") followed by the item text with inline formatting via renderFormattedText
   - Bullets use Helvetica 11pt, #333333 fill color, with 15pt indent for the bullet and 25pt indent for the text
   - 6pt space between items, 16pt space after section
   - Let PDFKit handle automatic page breaks (do NOT use explicit Y coordinates for body content)

6. **addPageNumbers(doc)** — After all content, use `doc.bufferedPageRange()` to iterate pages and add "Page X of Y" in Helvetica 9pt, #888888, centered at 50px from bottom. Use `doc.switchToPage(i)` for each page.

7. **generateSummaryPdf(summaryMarkdown: string, metadata: PdfMetadata): Promise<Buffer>** — Main export:
   - Create PDFDocument with: `size: "letter"`, margins 72 all sides, `bufferPages: true`, info.Title = "Executive Summary - {sourceFileName}", info.Author = "Telegram File Agent"
   - Set up Buffer collection: create chunks array, listen to `data` event BEFORE any content rendering, create Promise that resolves on `end` event with `Buffer.concat(chunks)`
   - Call renderHeader, then for each parsed section call renderSection
   - Call addPageNumbers after all content
   - Call `doc.end()`
   - Return the Buffer promise

Use structured logging with pino Logger parameter on generateSummaryPdf. Log: PDF generation start (with sourceFileName), PDF generation complete (with page count from bufferedPageRange, buffer size in bytes).

Important details from research:
- Use `doc.font("Helvetica")` not font file paths — built-in fonts only
- Set up data/end listeners BEFORE calling doc.end() to avoid race condition
- Enable `bufferPages: true` in constructor for page number injection
- Never write to filesystem — everything in-memory via Buffer
