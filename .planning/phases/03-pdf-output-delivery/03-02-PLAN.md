  <action>
Modify `src/handlers/documents.ts` to import and use the PDF generator after summarization.

**New imports:**
```typescript
import { InputFile } from "grammy";
import { generateSummaryPdf } from "../services/pdf-generator.js";
```

**Modify the document handler** (`message:document` handler) — after the existing text summary reply loop, add:

```typescript
// Generate and send PDF
const pdfBuffer = await generateSummaryPdf(summary, {
  sourceFileName: fileName,
  generatedAt: new Date(),
}, ctx.logger);

const pdfFileName = `summary-${fileName.replace(/\.[^.]+$/, "")}.pdf`;
await ctx.replyWithDocument(
  new InputFile(pdfBuffer, pdfFileName),
  { caption: `Executive summary of "${fileName}"` }
);
```

Place the PDF generation AFTER the text chunks are sent (so the user gets immediate text feedback while PDF generates). If PDF generation fails, log the error but do NOT crash the handler — the text summary was already sent successfully. Wrap the PDF section in its own try/catch:

```typescript
try {
  const pdfBuffer = await generateSummaryPdf(summary, { ... }, ctx.logger);
  await ctx.replyWithDocument(...);
  ctx.logger?.info({ fileName, pdfSize: pdfBuffer.length }, "PDF sent successfully");
} catch (pdfError) {
  ctx.logger?.error({ error: pdfError, fileName }, "PDF generation failed, text summary already sent");
  await ctx.reply("(PDF generation failed — text summary above is your summary)").catch(() => {});
}
```

**Modify the photo handler** (`message:photo` handler) — same pattern. Use a descriptive filename since photos don't have filenames:

```typescript
const pdfFileName = `summary-photo-${Date.now()}.pdf`;
const sourceLabel = `Photo (${width}x${height})`;
```

The caption should be: `Executive summary of photo (${width}x${height})`.

Wrap in the same try/catch pattern — text summary first, PDF as a bonus.

**Update the success log** for both handlers to include `pdfSent: true/false` so monitoring can track PDF generation success rate.

Keep the existing error handling for the overall handler (the outer try/catch) unchanged. The PDF try/catch is nested inside the success path only.
