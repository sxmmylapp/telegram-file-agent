   - skipLibCheck: true
   - forceConsistentCasingInFileNames: true

<task type="auto">
  <name>Task 2: Create config, logger, and context type modules</name>
  <files>src/config.ts, src/logger.ts, src/types.ts</files>
  <action>
1. Create src/config.ts:
   - Import "dotenv/config" for local dev
   - Helper function `requireEnv(name: string): string` that throws descriptive error with the var name
   - Export `config` object (as const):
     - BOT_TOKEN: requireEnv("BOT_TOKEN")
     - AUTHORIZED_USER_IDS: parse requireEnv("AUTHORIZED_USER_IDS") as comma-separated list of numbers. Use `.split(",").map(s => parseInt(s.trim(), 10)).filter(id => !isNaN(id))`. Throw if the resulting array is empty.
     - NODE_ENV: process.env.NODE_ENV || "development"
     - LOG_LEVEL: process.env.LOG_LEVEL || "info"

2. Create src/logger.ts:
   - Import pino
   - Export `createLogger()`: returns a Pino logger
     - level from config or env (LOG_LEVEL), default "info"
     - In dev (NODE_ENV !== "production"): use pino-pretty transport with colorize, translateTime "yyyy-mm-dd HH:MM:ss"
     - In production: no transport (raw JSON)
     - base: { service: "telegram-file-agent" }
   - Export `createRequestLogger(parent: pino.Logger, updateId: number, userId?: number)`: returns child logger with requestId: `upd-${updateId}` and userId

3. Create src/types.ts:
   - Import Context from "grammy"
   - Import FileFlavor from "@grammyjs/files"
   - Import Logger type from "pino"
   - Define interface BotContextFlavor with: logger: Logger, requestId: string
   - Export type BotContext = FileFlavor&lt;Context&gt; & BotContextFlavor
  </action>

Verify src/config.ts throws when BOT_TOKEN is missing: `BOT_TOKEN="" tsx -e "import './src/config.ts'"` should fail with "Missing required environment variable: BOT_TOKEN".
  </verify>

All three compile cleanly.
  </done>
