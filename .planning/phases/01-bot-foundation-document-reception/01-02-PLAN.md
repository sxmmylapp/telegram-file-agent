  <files>src/middleware/auth.ts, src/middleware/logging.ts, src/handlers/commands.ts</files>
  <action>
1. Create src/middleware/auth.ts:
   - Export `createAuthMiddleware(authorizedUserIds: number[]): Middleware&lt;BotContext&gt;`
   - Check `ctx.from?.id` against the authorizedUserIds ARRAY (use `.includes()`)
   - If unauthorized: log WARN with { attemptedUserId, chatId }, reply "This bot is private. Access denied.", return (do NOT call next())
   - If authorized: call `await next()`
   - Handle edge case: if `ctx.from` is undefined (channel posts, etc.), reject

2. Create src/middleware/logging.ts:
   - Export `createLoggingMiddleware(parentLogger: pino.Logger): Middleware&lt;BotContext&gt;`
   - Generate requestId from `ctx.update.update_id`: format "upd-{updateId}"
   - Create child logger via `createRequestLogger(parentLogger, updateId, ctx.from?.id)`
   - Attach to `ctx.logger` and `ctx.requestId`
   - Log DEBUG at start: "Processing update" with update type info
   - Call `await next()`
   - Log DEBUG after: "Update processed" with duration (measure with Date.now() before/after next())

3. Create src/handlers/commands.ts:
   - Create a `Composer&lt;BotContext&gt;` and export as `commandHandlers`
   - /start: Reply with welcome message including bot name and description ("I'm your real estate document assistant. Send me a PDF or image and I'll help you process it.")
   - /help: Reply with available commands (/start, /help, /status) and instructions for sending documents
   - /status: Reply with bot status (uptime via process.uptime(), version from package.json import)
   - Fallback `on("message:text")`: Reply with "I don't understand that command. Try /help or send me a document." -- this makes the bot conversational per BOT-01
   - Log INFO for each command with command name
  </action>

<task type="auto">
  <name>Task 2: Create document handlers and wire bot.ts with error handling and graceful shutdown</name>
  <files>src/handlers/documents.ts, src/bot.ts</files>
  <action>
1. Create src/handlers/documents.ts:
   - Create a `Composer&lt;BotContext&gt;` and export as `documentHandlers`
   - Define MAX_FILE_SIZE = 20 * 1024 * 1024 (20MB)
   - Helper: `formatBytes(bytes: number): string` -- returns human-readable size (B, KB, MB, GB). If bytes is 0, return "unknown size".

   - Handler for "message:document":
     - Extract file_name (default "unnamed"), file_size (default 0 via ?? operator), mime_type (default "unknown")
     - Log INFO: { fileName, fileSize, mimeType } "Document received"
     - If file_size is 0: warn user "Could not determine file size. Proceed with caution."
     - If file_size > MAX_FILE_SIZE: reply with warning including file name, actual size, and the 20MB limit. Return early.
     - Otherwise: reply with document details:
       ```
       Received document:
       - Name: {fileName}
       - Size: {formattedSize}
       - Type: {mimeType}

       Ready to process this file?
       ```
     - Log DEBUG with full document metadata

   - Handler for "message:photo":
     - Get the LAST element from ctx.msg.photo array (highest resolution)
     - Extract file_size (default 0), width, height
     - Log INFO: { fileSize, width, height } "Photo received"
     - If file_size > MAX_FILE_SIZE: warn about 20MB limit, suggest sending as uncompressed document. Return early.
     - Otherwise: reply with photo details:
       ```
       Received photo:
       - Size: {formattedSize}
       - Dimensions: {width}x{height}
       - Type: JPEG (Telegram-compressed)

       Ready to process this image?
       ```

2. Create src/bot.ts (main entry point):
   - Import config, createLogger, createAuthMiddleware, createLoggingMiddleware, commandHandlers, documentHandlers
   - Import Bot from "grammy", hydrateFiles from "@grammyjs/files", autoRetry from "@grammyjs/auto-retry"
   - Import BotContext type

   - Create logger via createLogger()
   - Create bot via `new Bot&lt;BotContext&gt;(config.BOT_TOKEN)`

   - Install API-level plugins:
     - `bot.api.config.use(hydrateFiles(bot.token))`
     - `bot.api.config.use(autoRetry())`

   - Global error handler via `bot.catch()`:
     - Import GrammyError, HttpError from "grammy"
     - Log ERROR with full error object (err.error), update_id, and error type
     - Discriminate: if GrammyError log "Telegram API error", if HttpError log "HTTP error", else log "Unknown error"
     - Reply to user: "Something went wrong. Please try again." (wrap in .catch(() => {}) to avoid double-error)
     - This satisfies BOT-04

   - Register middleware in order:
     1. createLoggingMiddleware(logger) -- FIRST so every update gets logged
     2. createAuthMiddleware(config.AUTHORIZED_USER_IDS) -- SECOND so auth is checked before handlers
     3. commandHandlers
     4. documentHandlers

   - Graceful shutdown:
     - process.once("SIGINT", () => { logger.info("Received SIGINT, stopping..."); bot.stop(); })
     - process.once("SIGTERM", () => { logger.info("Received SIGTERM, stopping..."); bot.stop(); })

   - Start bot:
     ```typescript
     bot.start({
       onStart: (botInfo) => {
         logger.info({ username: botInfo.username, authorizedUsers: config.AUTHORIZED_USER_IDS }, "Bot started");
       },
     });
     ```
